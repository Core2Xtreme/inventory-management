Backend Architecture

Filter Pipeline

  • apply_filters()
    Chains warehouse → category → status (AND logic, case-insensitive, skips "all")
  • filter_by_month()
    Supports YYYY-MM direct or Q1/Q2/Q3/Q4-YYYY quarter aliases
  • Inventory exception
    No time dimension in inventory.json — month filter not applied

Global Filter System

4 Filters — Applied Across All Views

  • Time Period ?month=
    all · 2025-01 through 2025-12 · Q1–Q4-2025 aliases
    Applied to: Orders, Dashboard Summary only
  • Location / Warehouse ?warehouse=
    all · San Francisco · London · Tokyo
    Applied to: Inventory, Orders, Dashboard
  • Category ?category=
    all · circuit boards · sensors · actuators · controllers · power supplies
    Applied to: Inventory, Orders, Dashboard
  • Order Status ?status=
    all · delivered · shipped · processing · backordered
    Applied to: Orders, Dashboard only

Filter Rules

  • All filters AND together — all must match for an item to appear
  • Value "all" skips that filter entirely
  • Matching is case-insensitive on the backend
  • Inventory has no time dimension — month filter is never sent to /api/inventory
  • Filter state lives in singleton useFilters() — shared across all views
  • Views watch filter changes and call loadData() reactively
  • Revenue goals: $800K/month single · $9.6M YTD all months
Key Patterns & Conventions

Frontend Loading Pattern

  • loading ref — shows spinner
  • error ref — shows error state
  • data ref — raw API response
  • computed() — derived/filtered data
  • try/catch/finally in every loadData()

v-for Key Rule

  • Always use unique IDs: :key="item.id"
  • Never use array index as key
  • For months: :key="item.month"
  • For inventory: :key="item.sku"
  • Prevents Vue reconciliation/rendering bugs

Backend Filter Chain

  • Query params parsed by FastAPI automatically
  • apply_filters() — warehouse, category, status
  • filter_by_month() — month or quarter
  • Pydantic response_model validates output
  • HTTPException(404) for missing items
